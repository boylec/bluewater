# Install

Bluewater is a [Gas City](https://github.com/gastownhall/gascity) pack.
You install it the same way as any other pack: with `gc pack add`, or by
declaring an `[imports.<name>]` entry in your `city.toml`.

## Prerequisites

- **Gas City `>= 1.0.0`.** Pack V2 is mandatory; v0.x layouts are not
  supported.
- **A beads provider.** `bd` (default) or `file`.
- **Standard Gas City tooling.** `tmux`, `git`, `jq`, `pgrep`, `lsof`.
- **Two or more LLM providers configured.** Cross-provider concurrence
  on two-key launches assumes at least two of `claude`, `codex`,
  `gemini`.

## Install — minimum running ship

The umbrella pack contains the wardroom plus all six required
departments inline (Engineering, Operations, Combat Systems, Supply,
Medical, Admin). One import gives you a fully functional ship:

```bash
gc pack add github.com/boylec/bluewater
```

## Install — optional add-ons

| Sub-pack             | When to add                                          |
|----------------------|------------------------------------------------------|
| `bluewater-air`      | You do production deploys and want choreographed flight-deck workflow |
| `bluewater-discord`  | You want Discord intake (Quarterdeck Watch)         |
| `bluewater-slack`    | You want Slack intake (Quarterdeck Watch)           |

```bash
gc pack add github.com/boylec/bluewater/packs/bluewater-air
gc pack add github.com/boylec/bluewater/packs/bluewater-discord
gc pack add github.com/boylec/bluewater/packs/bluewater-slack
```

You can install both Discord and Slack — two Quarterdeck Watch
instances coexist; both route to the same watchstander pool.

## Configure providers

Bluewater ratings are pre-stamped with model tiers (Strategic/Tactical/
Operational/Worker) that map onto Opus/Sonnet/Haiku. Provider per-
rating is set so two-key launches naturally span providers:

| Officer / role | Default provider | Default model tier |
|----------------|------------------|--------------------|
| CO             | claude           | opus               |
| XO             | codex            | sonnet             |
| COB            | gemini           | opus               |
| OOD            | claude           | sonnet             |
| CSOOW          | codex            | sonnet             |
| MA             | codex            | sonnet             |
| YN             | claude           | sonnet             |
| JAG            | gemini           | sonnet             |

If your ship has only one provider configured, the install-time check
(below) emits a warning; two-key features that require cross-provider
concurrence are degraded to single-provider concurrence (logged
prominently).

## Verify install

```bash
gc doctor
```

Expected output includes:

- `[ok] bluewater pack imported`
- `[ok] battle_bill.toml present (24 evolutions, 29 watchstations bound)`
- `[ok] watchstation_bindings.toml lints clean`
- `[ok] doctrine/brevity template fragments resolve`
- `[warn] only one provider configured` — only if you have one provider
